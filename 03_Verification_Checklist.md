# 03_Verification_Checklist — ガイドライン有効性の検証

本ドキュメントは、プロジェクト（例: 車載 IoT アクチュエータ）において**力学、用途定義、BOM、テストケース、テスト計測器、出荷前許認可**が定義されたとき、このガイドラインが有効であることをチェックするための検証リストである。

---

## 検証の目的

設計依存カスケードに沿って、以下が**定義済み**かつ**トレーサブル**であることを確認する。

```
[力学] + [Industry] → [用途] + [BOM] → [テストケース] → [テスト計測器] → [流通予定国] → [出荷前許認可]
```

---

## 検証チェックリスト

### 1. 力学スクリーニング

| # | チェック項目 | 確認 | 備考 |
|---|-------------|------|------|
| 1.1 | 適用ドメイン（E/M/T/F）が明示されている | ☐ | 00_Screening § 適用ドメイン |
| 1.2 | 適用外ドメインが定義されている | ☐ | 空力等、本プロジェクトで不要なドメイン |
| 1.3 | ドメイン境界の例外（あれば）が記載されている | ☐ | |
| 1.4 | 力学の選択根拠が用途と整合している | ☐ | 例: LTE-M 使用 → E 必須 |

### 2. Cross Industry Screening

| # | チェック項目 | 確認 | 備考 |
|---|-------------|------|------|
| 2.1 | 産業種別が選択されている | ☐ | 車載 IoT / EV / スマートフォン / ロケット / 大型量子観測施設 / 量子コンピュータ / 重力波観測施設 / ミクロ観測施設 / 宇宙有人・探査機 等 |
| 2.2 | 産業別追加要件が適用されている | ☐ | 00_Screening § Cross Industry Screening |
| 2.3 | 力学 + Industry から用途・BOM への導出が説明可能 | ☐ | 設計依存カスケードの第 1 段 |

### 3. 用途定義（Use Case）

| # | チェック項目 | 確認 | 備考 |
|---|-------------|------|------|
| 3.1 | 運用シーンが記載されている | ☐ | 例: レンタカー事業での車両載置型運用 |
| 3.2 | 設置環境・寸法拘束が定義されている | ☐ | 例: 杯架 45–75 mm 径 |
| 3.3 | 電源条件・待機時間が定義されている | ☐ | 例: 72 時間バッテリー待機 |
| 3.4 | 熱環境が定義されている | ☐ | 例: -40 °C ～ +85 °C |
| 3.5 | 用途が力学・Industry と整合している | ☐ | 矛盾がないこと |

### 4. BOM

| # | チェック項目 | 確認 | 備考 |
|---|-------------|------|------|
| 4.1 | 全部品に Mass / Cost / Lead Time / Alternate が記載されている | ☐ | BOM Governance |
| 4.2 | カテゴリ（通信制御、電源、物理駆動等）が付与されている | ☐ | 推奨 |
| 4.3 | 主要スペック・選定理由が記載されている | ☐ | 推奨 |
| 4.4 | BOM が用途定義を満たす構成になっている | ☐ | 例: LTE-M → nRF9160 等 |
| 4.5 | 用途 + BOM からテストケースへの導出が説明可能 | ☐ | 設計依存カスケードの第 2 段 |

### 5. テストケース

| # | チェック項目 | 確認 | 備考 |
|---|-------------|------|------|
| 5.1 | 各力学ドメインに対応するテストが定義されている | ☐ | E→EMC, M→振動, T→耐熱 等 |
| 5.2 | 産業別必須テストが含まれている | ☐ | 車載 IoT: 接続性、トルク、熱安定性 |
| 5.3 | 用途・BOM の主要機能をカバーするテストがある | ☐ | トレーサビリティ |
| 5.4 | テストケースからテスト計測器への導出が説明可能 | ☐ | 設計依存カスケードの第 3 段 |
| 5.5 | 環境試験の記述形式が統一されている | ☐ | IP コード、IEC 60068 等 |

### 6. テスト計測器

| # | チェック項目 | 確認 | 備考 |
|---|-------------|------|------|
| 6.1 | 各テストケースに対応する計測器・設備が特定されている | ☐ | トレーサビリティ |
| 6.2 | 計測器の精度・仕様がテスト要求を満たす | ☐ | |
| 6.3 | 外部委託が必要なテストと機器が明確である | ☐ | EMC 試験所等 |
| 6.4 | テストケースを実施するための機器が不足していない | ☐ | 網羅性 |

### 7. 流通予定国（許認可の前に定義）

| # | チェック項目 | 確認 | 備考 |
|---|-------------|------|------|
| 7.1 | 流通予定国が国・地域単位で列挙されている | ☐ | 例: 日本、米国、ドイツ、フランス |
| 7.2 | 段階的流通（初期・後期）がある場合は記載されている | ☐ | 例: 初期＝日本・米国、後期＝EU 加盟国 |
| 7.3 | 流通予定国が許認可の**前**に定義されている | ☐ | 流通先を決めた上で該当国の認証を取得 |

### 8. 出荷前許認可

| # | チェック項目 | 確認 | 備考 |
|---|-------------|------|------|
| 8.1 | 流通予定国に応じた法令が参照されている | ☐ | 02_Regulatory_Compliance |
| 8.2 | 力学・Industry に応じた法令が参照されている | ☐ | |
| 8.3 | 無線機能がある場合、FCC / 電波法 / RED の該当が確認されている | ☐ | |
| 8.4 | 電気用品該当の場合、PSE の該当が確認されている | ☐ | |
| 8.5 | 許認可の取得状況または取得計画が記載されている | ☐ | |

### 9. 設計依存カスケードの整合性

| # | チェック項目 | 確認 | 備考 |
|---|-------------|------|------|
| 9.1 | 力学 + Industry → 用途・BOM の順序が守られている | ☐ | 逆順で定義していない |
| 9.2 | 用途・BOM → テストケース の順序が守られている | ☐ | |
| 9.3 | テストケース → テスト計測器 の順序が守られている | ☐ | |
| 9.4 | 流通予定国 → 出荷前許認可 の順序が守られている | ☐ | 流通先を決めてから該当国の認証を取得 |
| 9.5 | 許認可がスクリーニング完了後に参照されている | ☐ | |

## 例: 車載 IoT アクチュエータ（Tpod 相当）の検証例

| 項目 | 定義例 | ガイドラインとの整合 |
|-----|--------|---------------------|
| **力学** | E, M, T（F は適用外） | 00_Screening § 適用ドメイン |
| **Industry** | 車載 IoT | Cross Industry Screening |
| **用途** | 杯架設置、LTE-M 遠隔操作、72 h 待機、-40～+85 °C | ユースケース・運用コンテキスト |
| **BOM** | nRF9160, LiFePO4, マイクロサーボ, NTC, PC/PA12 筐体 | BOM Governance, RFQ-to-Design Gap |
| **テストケース** | 接続性、トルク、熱安定性、EMC、耐熱 | 01_Hardware_Mass_Production_Model, RFQ |
| **テスト計測器** | LTE-M 試験環境、トルク計、恒温槽、EMC 試験所 | 設計依存カスケード |
| **流通予定国** | 日本、米国、ドイツ（初期）；EU 他国（後期） | 許認可の前に定義 |
| **出荷前許認可** | FCC Part 22/27（US）, 技適（Japan）, RED（EU） | 02_Regulatory_Compliance |

---

## 検証結果の判定

- **全項目 ☑**: ガイドラインに沿った定義が完了している。有効性あり。
- **未確認あり**: 該当項目を定義・更新し、再検証すること。
- **トレーサビリティ断裂**: 設計依存カスケードの順序を確認し、逆順やスキップを修正すること。
